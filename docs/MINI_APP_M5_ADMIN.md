# Mini App M5 — администратор MVP-1

## Что готово

- Защищённая очередь администратора доступна только Telegram-аккаунту из `ADMIN_TELEGRAM_ID`.
- В очереди есть две вкладки: заявки, требующие решения, и недавние результаты.
- Карточка заявки показывает пользователя, формат, дату, длительность, комментарий и технический статус.
- Поддержаны действия `confirm`, `reject` и `block`; для отклонения и блокировки можно указать причину.
- Перед подтверждением сервер повторно проверяет доступность слота.
- Повторное или конкурентное решение не создаёт второе событие: API возвращает текущий результат `ALREADY_PROCESSED` либо общий результат уже выполняющегося решения.
- Отдельно показаны понятные результаты: слот уже занят, ошибка Google Calendar, заявка уже обработана другим администратором.
- При блокировке пользователь получает статус `BANNED`, его ожидающие заявки отклоняются, а резервы времени освобождаются.

## API

Все маршруты требуют Mini App session cookie. Записывающие запросы дополнительно проверяют `Origin`.

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/bookings?scope=pending` | Очередь заявок и сводные счётчики |
| `GET` | `/api/mini-app/v1/admin/bookings?scope=recent` | Недавно обработанные заявки |
| `GET` | `/api/mini-app/v1/admin/bookings/:id` | Полная карточка заявки |
| `POST` | `/api/mini-app/v1/admin/bookings/:id/confirm` | Повторная проверка слота и подтверждение |
| `POST` | `/api/mini-app/v1/admin/bookings/:id/reject` | Отклонение с необязательной причиной |
| `POST` | `/api/mini-app/v1/admin/bookings/:id/block` | Отклонение и блокировка пользователя |

Тело для `reject` и `block`:

```json
{
  "reason": "Необязательное пояснение до 500 символов"
}
```

## Результаты решения

- `CONFIRMED` — встреча подтверждена, существует ровно одно Calendar-событие.
- `REJECTED` — заявка отклонена, резерв снят.
- `BLOCKED` — заявка отклонена, пользователь заблокирован.
- `SLOT_UNAVAILABLE` — слот успел стать недоступным, событие не создано.
- `CONFIRMATION_ERROR` — Google Calendar не подтвердил операцию; заявка остаётся видимой для безопасного завершения.
- `ALREADY_PROCESSED` — другой администратор или повторный запрос уже принял решение.
