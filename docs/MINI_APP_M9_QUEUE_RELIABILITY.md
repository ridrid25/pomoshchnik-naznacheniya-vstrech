# Mini App M9 — надёжное согласование

## Основание

Пилот M8 завершён на 9 настоящих заявках. Две заявки, или 22% пилотного потока, перешли в `SLOT_UNAVAILABLE`: время было свободно при отправке, но стало занято до решения администратора.

## Цель M9

Сделать очередь согласования предсказуемой: администратор до нажатия решения видит актуальное состояние слота и не запускает заведомо невозможное подтверждение.

## Пакет M9.1 — живая проверка времени

- API очереди и карточки заявки повторно проверяет рабочее расписание, активные резервы, подтверждённые встречи и Google Calendar;
- собственный резерв проверяемой заявки исключается из конфликта;
- для ожидающей заявки API возвращает `slotAvailable`;
- `canConfirm=false`, если время уже занято;
- в очереди и подробной карточке показано «Время свободно — можно подтверждать» либо «Время уже занято — подтверждение недоступно»;
- для занятого времени остаётся доступно отклонение заявки;
- серверная проверка при подтверждении сохраняется и защищает от конфликта между просмотром и нажатием.
- если Telegram передал подписанные данные запуска в адресе, но клиентский SDK не загрузился, Mini App использует эти данные для создания сессии; подпись и срок действия по-прежнему проверяет backend.

## Контрольная точка M9.1

1. Свободная ожидающая заявка показывает положительный статус и кнопки подтверждения и отклонения.
2. Если в Google Calendar или расписании появляется конфликт, очередь после обновления показывает занятость.
3. Для занятого слота кнопка подтверждения отсутствует, а отклонение остаётся доступным.
4. Прямой запрос подтверждения всё равно повторно проверяет занятость на сервере.
5. Production health, Telegram menu button и Mini App остаются рабочими.
6. Запуск во встроенном Telegram Web работает даже при недоступности клиентского Telegram SDK.
