# Telegram Mini App: контрольная точка M2

M2 — интерактивный mobile-first дизайн-прототип на mock-данных. Он не подключён к API, не попадает в Docker-образ и не меняет production-маршрутизацию.

## Что можно проверить

- Главная с ближайшей встречей, активными заявками и основной кнопкой записи.
- Пошаговый мастер: детали → дата и время → контакты → проверка → успех.
- Форматы «Онлайн» с Google Meet и «Личная» без видеоссылки.
- Длительности 30, 45 и 60 минут, где 30 минут выбраны по умолчанию.
- Telegram как основной канал ответа и необязательный email для Calendar invite.
- Раздел «Мои записи»: активные заявки и архив.
- Экран администратора: очередь согласования, подтверждение, отклонение и блокировка.
- Светлая и тёмная темы, размеры 320, 360, 390 и 430 px.

В прототипе нет фотографии, аватара и приветственной фразы. Визуальное направление: насыщенные индиго/фиолетовый и бирюзовый, точечный тёплый акцент, объёмные поверхности и спокойная анимация.

## Локальный запуск

```powershell
npm.cmd run prototype:serve
```

Открыть `http://127.0.0.1:4173`. Для принудительной темы можно использовать `?theme=light` или `?theme=dark`.

## Автоматическая проверка

```powershell
npm.cmd run prototype:smoke
```

Проверка контролирует наличие всех согласованных экранов и продуктовых решений, отсутствие внешних ресурсов и изображений, синтаксис JavaScript, отдачу локальных файлов и защиту тестового static server от выхода за каталог прототипа.

## Граница этапа

Сейчас все данные демонстрационные. До ручного визуального согласования прототипа нельзя подключать рабочий API. После согласования этап M3 переносит дизайн в пользовательский поток записи и связывает его с существующими M1 endpoints.
