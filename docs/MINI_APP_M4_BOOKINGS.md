# Telegram Mini App: контрольная точка M4

Этап M4 подключает личный раздел пользователя к существующим данным бота и реализует отмену, перенос и выбор канала уведомлений.

## Пользовательский сценарий

- «Мои записи» разделены на активные и архив.
- Активными считаются заявки `PENDING_APPROVAL`, `CONFIRMATION_ERROR` и будущие подтверждённые встречи.
- В архив попадают отклонённые, истёкшие, отменённые, недоступные и завершённые встречи.
- Карточка показывает номер, статус, дату, время, длительность, формат, комментарий, email и Google Meet, если ссылка уже создана.
- Отменить можно ожидающую или будущую подтверждённую встречу.
- Перенести можно только будущую подтверждённую встречу.
- При переносе старая встреча остаётся `CONFIRMED`, а новое время создаётся отдельной заявкой `RESCHEDULE / PENDING_APPROVAL`.
- Ответы можно получать в Telegram или на сохранённый email.

## API

- `GET /api/mini-app/v1/bookings?scope=active|archive` — список собственных записей.
- `GET /api/mini-app/v1/bookings/:id` — подробности собственной записи.
- `POST /api/mini-app/v1/bookings/:id/cancel` — повторяемо-безопасная отмена.
- `POST /api/mini-app/v1/bookings/:id/reschedule` — запрос переноса с новым `startAt` и `idempotencyKey`.
- `PATCH /api/mini-app/v1/me/notifications` — канал `TELEGRAM | EMAIL` и, при необходимости, email.

Все изменяющие запросы требуют одновременно session cookie и точный `Origin`, соответствующий `PUBLIC_BASE_URL`. Чужая запись возвращается как несуществующая.

## Гарантии

- Повторная отмена возвращает уже отменённую запись и не отправляет повторные уведомления.
- Повтор переноса с тем же `idempotencyKey` возвращает ранее созданную заявку.
- Новое время переноса повторно проверяется через `AvailabilityService` на сервере.
- При отмене освобождается резерв; существующее Calendar-событие отменяется через прежний `BookingService`.
- Пользователь и администратор получают уведомление через существующий `NotificationService`.

## Проверка

```powershell
npm.cmd run typecheck
npm.cmd run prototype:smoke
npm.cmd run test:e2e
npm.cmd test
```

M4 E2E проверяет списки, карточку, переключение уведомлений, перенос подтверждённой встречи без изменения исходной, идемпотентный повтор переноса, отмену, повтор отмены и переход отменённой записи в архив.
