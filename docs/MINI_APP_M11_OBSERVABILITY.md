# Mini App M11 — наблюдение и самодиагностика

Этап добавляет владельцу понятное наблюдение за помощником без технических терминов.

## Три части

1. **Состояние помощника**
   - новый раздел `Управление → Состояние помощника`;
   - ручная проверка приложения, данных, Telegram, Google Calendar, уведомлений, отметок календаря и очереди заявок;
   - три понятных состояния: «Всё работает», «Нужно внимание», «Есть неисправность».

2. **Безопасное самовосстановление**
   - каждые пять минут production проверяет основные связи;
   - повторяются только ожидающие отправки уведомления;
   - восстанавливаются отсутствующие отметки заявок на согласовании в Google Calendar;
   - при необходимости восстанавливается правильный webhook Telegram без удаления накопившихся сообщений;
   - заявки не подтверждаются, не отклоняются и не удаляются автоматически;
   - при проблеме владелец получает сообщение в Telegram, а после восстановления — сообщение о нормальной работе.

3. **Отчёт для Codex**
   - кнопка «Скопировать отчёт для Codex» создаёт готовый текст;
   - в отчёт входят время проверки, версия, состояние разделов и уже выполненные попытки восстановления;
   - токены, пароли, email, Telegram ID и содержание заявок в отчёт не попадают.

## Защищённый API

| Метод | Адрес | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/diagnostics` | Проверить состояние без изменений |
| `POST` | `/api/mini-app/v1/admin/diagnostics/repair` | Запустить безопасное восстановление |

Оба адреса требуют действующую сессию владельца. Изменяющий запрос дополнительно проверяет Origin.

## Контроль сохранности прежних функций

M11 не меняет существующие точки входа Telegram-бота, запись в чате, пользовательские заявки, очередь согласования, ссылки возврата из Google Calendar, настройки занятости и архив прошедших встреч.
