# Mini App M10 — центр управления

После завершения M9 следующий продуктовый этап закрывает оставшиеся разделы администраторской части MVP-2.

## Пакет M10.1 — закрытые даты и интервалы

- из экрана «Настройки» открывается отдельный раздел «Закрытые даты»;
- администратор может закрыть весь будущий день или конкретный интервал;
- доступен необязательный внутренний комментарий до 500 символов;
- одинаковое ограничение не создаётся повторно;
- будущую блокировку можно удалить с подтверждением;
- ограничения сразу участвуют в общем расчёте свободных окон бота и Mini App;
- список и действия доступны только Telegram-администратору, а изменения защищены проверкой `Origin`.

## API

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/restrictions` | Будущие ограничения и часовой пояс расписания |
| `POST` | `/api/mini-app/v1/admin/restrictions` | Закрыть день или интервал |
| `DELETE` | `/api/mini-app/v1/admin/restrictions/:id` | Удалить будущее ограничение |

## Контрольная точка M10.1

1. Обычный пользователь не получает список ограничений.
2. Запрос с посторонним `Origin` отклоняется.
3. После закрытия дня API доступности возвращает для него ноль окон.
4. Повторная отправка не создаёт дубль.
5. После удаления время снова рассчитывается по основному расписанию и Google Calendar.

## Пакет M10.2 — заблокированные пользователи

- в центре управления доступен отдельный список активных блокировок;
- карточка показывает имя, Telegram username, дату и причину блокировки;
- создать блокировку по-прежнему можно только из конкретной заявки;
- разблокировка выполняется после явного подтверждения;
- повторный запрос разблокировки безопасен и возвращает `changed=false`;
- после разблокировки пользователь снова может открыть Mini App и отправлять заявки.

## API M10.2

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/blocked-users` | Активные блокировки |
| `POST` | `/api/mini-app/v1/admin/blocked-users/:userId/unblock` | Безопасная разблокировка |

## Контрольная точка M10.2

1. Обычный пользователь не получает список блокировок.
2. В списке нет внутренних Telegram ID, email и данных чужих заявок.
3. Запрос с посторонним `Origin` отклоняется.
4. После разблокировки запись исчезает из списка, а пользователь снова проходит авторизацию.
5. Повторное действие не изменяет данные повторно.

## Пакет M10.3 — шаблоны уведомлений

- из центра управления открывается список всех восьми системных уведомлений;
- каждое уведомление редактируется на отдельном мобильном экране;
- редактор показывает только доступные для выбранного события подстановки;
- нажатие на подстановку вставляет её в позицию курсора;
- сервер отклоняет пустой текст, текст длиннее 4000 символов, неизвестные переменные и непарные фигурные скобки;
- изменения сразу используются для следующих Telegram- и email-уведомлений.

Подстановки с датой, временем, часовым поясом, длительностью и форматом доступны для отправленной и подтверждённой заявки. Для отклонения доступна необязательная причина. В событиях, где вызывающий процесс не передаёт переменные гарантированно, используются только обычные тексты.

## API M10.3

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/templates` | Список текстов и допустимых подстановок |
| `PATCH` | `/api/mini-app/v1/admin/templates/:type` | Проверить и сохранить текст шаблона |

## Контрольная точка M10.3

1. Обычный пользователь не получает шаблоны.
2. Запрос изменения с посторонним `Origin` отклоняется.
3. Неизвестная подстановка не сохраняется.
4. Допустимые подстановки сохраняются и возвращаются без изменения.
5. В интерфейсе есть отдельные список и редактор, пригодные для экрана телефона.

## Пакет M10.4 — рабочая неделя

- администратор видит все семь дней недели, включая выходные;
- любой день можно включить или выключить одним переключателем;
- в одном дне поддерживается до четырёх интервалов, например до и после обеда;
- время выбирается с шагом 15 минут, один интервал длится не меньше 30 минут;
- клиент заранее предупреждает о пересечениях и расписании без рабочих дней;
- сервер повторяет все проверки и заменяет расписание атомарно вместе с основными правилами записи;
- изменения сразу участвуют в расчёте свободных дат и окон Telegram-бота и Mini App.

## API M10.4

`PATCH /api/mini-app/v1/admin/settings/schedule` принимает поле `workingPeriods`:

```json
[
  { "weekday": 1, "startMinute": 540, "endMinute": 780 },
  { "weekday": 1, "startMinute": 840, "endMinute": 1080 }
]
```

Если поле не передано, текущее недельное расписание сохраняется — это оставляет API совместимым с предыдущими клиентами.

## Контрольная точка M10.4

1. Можно включать и выключать рабочие дни.
2. Можно создать несколько непересекающихся интервалов одного дня.
3. Пересекающиеся, слишком короткие и не кратные 15 минутам интервалы отклоняются.
4. Нельзя случайно сохранить расписание без единого рабочего интервала.
5. Основные правила и рабочие интервалы сохраняются одной транзакцией.

## Итог M10

Этап M10 закрыт полностью. В Mini App доступны состояние Google Calendar, основные правила записи, часовые пояса России, рабочая неделя, закрытые даты и интервалы, список заблокированных пользователей и безопасный редактор шаблонов уведомлений.
