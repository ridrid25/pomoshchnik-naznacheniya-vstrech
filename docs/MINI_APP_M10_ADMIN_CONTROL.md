# Mini App M10 — центр управления

После завершения M9 следующий продуктовый этап закрывает оставшиеся разделы администраторской части MVP-2.

## Пакет M10.1 — закрытые даты и интервалы

- из экрана «Настройки» открывается отдельный раздел «Закрытые даты»;
- администратор может закрыть весь будущий день или конкретный интервал;
- доступен необязательный внутренний комментарий до 500 символов;
- одинаковое ограничение не создаётся повторно;
- будущую блокировку можно удалить с подтверждением;
- ограничения сразу участвуют в общем расчёте свободных окон бота и Mini App;
- список и действия доступны только Telegram-администратору, а изменения защищены проверкой `Origin`.

## API

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/restrictions` | Будущие ограничения и часовой пояс расписания |
| `POST` | `/api/mini-app/v1/admin/restrictions` | Закрыть день или интервал |
| `DELETE` | `/api/mini-app/v1/admin/restrictions/:id` | Удалить будущее ограничение |

## Контрольная точка M10.1

1. Обычный пользователь не получает список ограничений.
2. Запрос с посторонним `Origin` отклоняется.
3. После закрытия дня API доступности возвращает для него ноль окон.
4. Повторная отправка не создаёт дубль.
5. После удаления время снова рассчитывается по основному расписанию и Google Calendar.

## Пакет M10.2 — заблокированные пользователи

- в центре управления доступен отдельный список активных блокировок;
- карточка показывает имя, Telegram username, дату и причину блокировки;
- создать блокировку по-прежнему можно только из конкретной заявки;
- разблокировка выполняется после явного подтверждения;
- повторный запрос разблокировки безопасен и возвращает `changed=false`;
- после разблокировки пользователь снова может открыть Mini App и отправлять заявки.

## API M10.2

| Метод | Маршрут | Назначение |
| --- | --- | --- |
| `GET` | `/api/mini-app/v1/admin/blocked-users` | Активные блокировки |
| `POST` | `/api/mini-app/v1/admin/blocked-users/:userId/unblock` | Безопасная разблокировка |

## Контрольная точка M10.2

1. Обычный пользователь не получает список блокировок.
2. В списке нет внутренних Telegram ID, email и данных чужих заявок.
3. Запрос с посторонним `Origin` отклоняется.
4. После разблокировки запись исчезает из списка, а пользователь снова проходит авторизацию.
5. Повторное действие не изменяет данные повторно.
